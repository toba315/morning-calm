## Description

アプリ起動からニュース一覧、詳細表示までの画面のつながりを線で結んだ図（フロー）を作る。

＞＞＞＞＞
*アプリケーション開発仕様書 兼 画面遷移図（テキスト版）
本ドキュメントは、スマートフォンの一般ユーザー向けにリリースする「非依存型・朝のメンタルチューニング 兼 習慣化アプリ」の開発仕様書である。
開発者は本要件および画面遷移ロジックに基づき、UIの実装およびバックエンド（生成AIを用いたデータ処理）の構築を行うこと。
・プロダクト名（仮）：Morning Calm（モーニング・カーム）
・対象プラットフォーム：モバイルアプリ（iOS / Android）

**デザインおよびビジュアルの要件
画面の約40%を余白とし情報が密集しないミニマリストUIとする 
ベースカラーは#F9F9F6（アイボリー）を使用し真っ白を避けて朝の目の負担を軽減する 
メインカラーは#E2ECE9（ペールミントグリーン）を使用し静けさと安心感を表現する 
アクセントカラーは#E67E22（ジェントル・オレンジ）を使用し連続ログイン等の強調箇所にのみ使用する 
テキストカラーは#2C3E50（ダークネイビー）を使用しコントラストを抑えて疲労を防ぐ 
フォントはOS標準のゴシック体（iOSはSF Pro、AndroidはRoboto）を使用する

**バックエンドおよびシステムロジックの要件
毎朝4:00に自動でWeb上のニュースを巡回しポジティブな客観的事実のみを選別する 
創造と誕生、進歩と克服、貢献と調和のいずれかに該当する事実のみを合格とする 
負の清算、対立と批判、過度な恐怖が含まれるニュースは一律で排除する 
選別した事実を元にAIが150文字以上200文字以内の完全オリジナル記事を新規執筆してDBに格納する ユーザーがアプリを開いたタイムスタンプを記録し日付を跨いで連続ログインしている日数を自動でカウントアップする 
電波が途切れても利用できるよう起動時にその日のデータを一括でローカル環境に保存するオフラインファースト設計とする

**画面遷移および各画面のUI仕様
スプラッシュ画面（共通の起動画面） 背景はアイボリーのグラデーションとする 
中央にアプリロゴをフェードイン表示する ロゴの下にアクセントカラーで連続ログイン日数を「〇日連続」と数字のみ静かに表示する 
3秒間のアニメーション終了後、無料ユーザーは無料版ニュース閲覧画面へ、有料ユーザーは有料版ニュース閲覧画面へ自動遷移する

***無料版ニュース閲覧画面 
1画面に1つのニュースカード（タイトルと本文）のみを大きく表示する 
本文は200文字以内のため縦スクロールの挙動を完全に無効化する 
画面最下部にノイズにならない静かな環境型広告を表示する 画面上部に残り本数を示すオレンジ色のミニマルなゲージを表示する 
ユーザーが画面を右から左へ指でスワイプすることで1本目から5本目へと進む導線とする 
前の記事に戻るための左から右への反転スワイプのみ許容する 
5枚目のカードをスワイプした瞬間に無料版今日の一覧画面へ自動遷移する

***有料版ニュース閲覧画面 
画面構成やスワイプ操作、ゲージ表示は無料版と同一とする 
無料版との違いとして画面最下部の環境型広告を100%非表示とし完全な聖域化を行う 
5枚目のカードをスワイプした瞬間に有料版今日の一覧・ログ画面へ自動遷移する

***無料版今日の一覧画面 
5本を読み終えた後に自動遷移する着地画面とする 
本日の5本のニュースのタイトルのみが縦にシンプルに並ぶリスト形式とする 
リスト内のタイトルをタップすると該当するニュースカードがポップアップで開き何度でも再読できる 
これ以上新しいニュースを追加読み込みするスクロール等の機能は一切持たせない 
画面下部に現在〇日連続達成中である旨と明日も朝の聖域でお会いしましょうというテキストを固定表示する

***有料版今日の一覧・ログ画面 
無料版と同様に5本のニュースのタイトルリストを表示しタップでの再読に対応する 
画面上部にミニマルなデザインの当月カレンダーを表示する 
5本読了した日はカレンダーの該当日の丸いスタンプがペールミントグリーンに優しく点灯し継続を可視化する 
カレンダー上でその日の気分（晴れ、曇り、雨）を1タップで重ねて記録できるUIとする 
各タイトルの横にしおりアイコンを配置し1タップでマイページへ無制限にストック保存できる機能を持たせる 
リストの最下部に次の5本をチューニングするというボタンを静かに配置する 
このおかわりボタンをタップすると新しい5本が用意され有料版ニュース閲覧画面へ再び遷移する 
おかわり機能は1日最大4セット（計20本）まで制限をかけ有料ユーザーであっても無限の依存を防ぐ設計とする

**開発実装時の注意点
ニュースカード画面における斜めスワイプの検知閾値を適切に設定し縦スクロールの制限と干渉して画面がカクつくストレスをゼロにすること 
通信同期のエラーによってユーザーの連続ログイン日数が誤ってリセットされないようローカル側でのカウント保持とサーバー同期の順序を徹底すること
